# Architecture Decision Records

ADRs document significant technical decisions, the context in which they were made, alternatives considered, and the consequences (good and bad). They are append-only — when a decision is superseded, mark the old ADR as such and create a new one.

Format follows [Michael Nygard's template](https://github.com/joelparkerhenderson/architecture-decision-record/blob/main/locales/en/templates/decision-record-template-by-michael-nygard/index.md), lightly extended with an "Alternatives considered" section.
